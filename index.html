    What's EzGame

    EzGame, or EaZy Game Engine, is a python library designed for high level game development. With various inspirations such as PopCap's SexyAppFramework, LÖVE or Pygame, its two main focuses are:

    • Reducing development time, complexity and make code easily re-usable with functionalities that are as high level as possible;
    • Benefit of all python's ecosystem, EzGame has builtin support for several libs and integrating others (such as image manipulation, graph drawing or even AI libs) can be done in seconds.

    Tech

    EzGame is designed for modern python, CPyhon 3.13+ is guaranteed to provide all functionalities. However, older versions from 3.8+ to 3.13 should work but may not be fully supported, use at your own risks. EzGame runs where CPython runs primarily, the engine is guaranted to fully support:

    • Windows 32-bit and 64-bit
    • MacOS 64-bit
    • Linux 64-bit

    The game packing tool (N.Y.I.) can also embed CPython to build projects for 2 additional platforms:

    • Android 64-bit
    • iOS 64-bit

    EzGame relies on SDL3 and addons for most features. It also uses OpenAL for audio, and implements an OpenGL rendering backend.

    Engine philosophy

    EzGame consists of a wide range of modules, from simple types and operations (ex.: ezgame.math) to advanced integrations (ex.: ezgame.discord, ezgame.steam). The (fake!) modules are actually hidden class instances, and therefore implement functionalities like properties, __getitem__ and __setitem__ and more:

    ezgame.window.size = (800, 600)

    size is a property, getting and setting it calls a function internally, but it's showcased as a member for highlevelness. Most getters and setters are properties in EzGame, functions are provided when real arguments are required. Along with the properties/methods, the modules provides classes, themselves following the same philosophy.

    A small example

    import ezgame
from ezgame.locals import BLUE
from ezgame.widget import Widget

class BlueBox(Widget):
    def __init__(self):
        self.rect = Rect(20, 20, 80, 80)

    def update(dt):
        ...

    def display():
        ezgame.canvas.fill(BLUE)

if __name__ == '__main__':
    ezgame.window.size = (800, 600)
    ezgame.widget.add(Box())
    ezgame.mainloop()

    This example opens a 800x600 window and displays a 80x80 blue square, at position 20,20. Let's explain each part:

    Widgets

    The main class used to interact with the engine is Widget, defined in ezgame.widget. While most pure-code engines use one game loop with update and draw function, Widgets in EzGame are sort of multiple game loops, each one defining update(dt), display(), and owning several properties such as a position, size, and layer for order of updating/rendering.

    A Widget can represent a simple button or even the entire game. In this example, the BlueBox widget is just constantly filled with blue and doesn't interact. For a widget to be active, it needs to be added into the game space from widget manager (ezgame.widget.add()).

    Mainloop

    To start the game, ezgame.mainloop() must be called. All widgets added in game space before the call will be active, and they can themselves add/remove widgets in their updating logic with the ezgame.widget manager. ezgame.quit() can also be called inside this logic to end the game, program flow will continue after the mainloop starting line. Quitting ezgame should generaly mean closing the app, since it destroys all widgets and most other objects related to the engine.

    For a large part of the modules functionalities, behaviour can differ depending on if mainloop was started. For example, the ezgame.window.size property just changes a value before mainloop, and changes window size in real time during mainloop. Some functionalities can also be exclusive to after/before mainloop.
